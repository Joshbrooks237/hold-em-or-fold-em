# HOLD ’EM OR FOLD ’EM

*Texas Hold’em don’t negotiate. Neither do I.*

This ain’t your grandma’s bridge club. This is **Cigar Room Hold’em** — a lean, mean, browser-based poker table built for folks who like their cards hot, their felt green, and their decisions final. You want fluff? Go knit a sweater. You want action? **Deal the hand.**

---

## What in the wide world of sports is this?

- **Six-handed Texas Hold’em** — you, five digital sharpies, one pot, zero excuses.
- **Full streets** — preflop through river. Blinds rotate. The button moves. The universe stays in balance.
- **Real hand math** — best five out of seven. No cheat codes. Mother Nature doesn’t bluff; neither does the evaluator.
- **Mobile-friendly** — plays on a phone without squinting like you’re reading the fine print on a government form.
- **Cigar-lounge vibe** — dark wood, gold trim, smoke on the water (metaphorically). Dress code: attitude.

Stack: **Vite**, **React**, **TypeScript**. Fast. Loud. American enough to run on coffee and spite.

---

## Stranglehold on your machine

### 1. Load the ammo

```bash
git clone https://github.com/Joshbrooks237/hold-em-or-fold-em.git
cd hold-em-or-fold-em
npm install
```

### 2. Crank it up

```bash
npm run dev
```

Open the URL the terminal spits out — usually `http://localhost:5173`. **Click “Next hand.”** Play tight. Play loose. Just don’t whine when the river runs bad.

### 3. Build for production (when you’re ready to ship)

```bash
npm run build
npm run preview
```

## House rules (the boring stuff that keeps the wheels on)

- **You** are seat one. The rest are bots with opinions and questionable taste in raises.
- **Showdown** reveals the goods; between hands, grab a breath and hit **Next hand** again.
- This is a **client-side** game — great for practice, vibes, and talking smack at your screen. Not a casino. Not legal advice. Don’t bet the farm on my code.
